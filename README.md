> "Experience the future of coding with Google Jules in VSCode"

Jules Extension is an extension that allows you to operate Google's AI coding agent **Jules** directly from within VSCode.
Welcome an intelligent partner to your coding workflow.

## ‚ú® Concept

This extension was created to take your development experience to the next level.

- **Seamless Integration:** Access the powerful features of Jules without leaving your usual VSCode environment.
- **Real-time Collaboration:** From creating a coding session to checking its progress, everything is in real-time.
- **Productivity Leap:** Leave the tedious tasks to Jules and focus on your creative work.

## üì¶ Installation

You can install the Jules Extension in two ways:

### From the Marketplace (Recommended)

1.  Search for "Jules Extension" in the VSCode Marketplace.
2.  Click the **Install** button.

### From a VSIX File (Manual Install)

If you want to try the latest features that are not yet published on the Marketplace, you can download and install the `.vsix` file directly from the releases page.

1. **Go to the Releases Page:**
   Visit [GitHub Releases](https://github.com/your-repo/jules-extension/releases) and find the latest release version.

2. **Download the VSIX File:**
   Download the `.vsix` file (e.g., `jules-extension-0.1.0.vsix`) from the `Assets`.

3. **Install in VSCode:**
   - Open VSCode.
   - Go to the `Extensions` view (`Ctrl+Shift+X`).
   - Click the `...` (More Actions) menu at the top of the view and select `Install from VSIX...`.
   - Select the downloaded `.vsix` file to install.

## üîë Getting Your API Key

To use the Jules Extension, you need a Jules API key. Follow these steps to get one:

1. **Create an Account:**

   - Go to the [Jules Official Website](https://jules.google/docs).
   - Sign up for a new account or log in if you already have one.

2. **Generate API Key:**

   - Navigate to the "API Keys" or "Developer Settings" section in your account dashboard.
   - Click on "Create a new secret key".
   - Give your key a descriptive name (e.g., "VSCode Extension") and generate it.

3. **Copy Your Key:**
   - Your new API key will be displayed. Copy it to your clipboard.
   - If you need to view your key again later, you can always find it on your Jules settings page.

> **Important:** Treat your API key like a password. Do not share it publicly or commit it to version control.

## Authentication for GitHub

To check the status of pull requests, the extension needs to authenticate with GitHub.

### OAuth Sign-in (Recommended) ‚úÖ

This is the recommended and most secure way to authenticate.

1.  Open the Command Palette (`Cmd+Shift+P` or `Ctrl+Shift+P`).
2.  Run the **`Jules: Sign in to GitHub`** command.
3.  You will be redirected to your browser to authorize the extension.

### GitHub PAT (Deprecated) ‚ö†Ô∏è

Using a Personal Access Token (PAT) is deprecated and will be removed in a future version. Please migrate to the OAuth sign-in method for better security and a smoother experience.

## ‚öôÔ∏è Extension Settings

This extension contributes the following settings. You can configure them in VSCode's `settings.json` file or through the settings UI.

| Setting | Description | Default |
| :--- | :--- | :--- |
| `jules-extension.apiKey` | Your Jules API key for authentication. This is stored securely in VSCode's secret storage. | `""` |
| `jules-extension.autoRefresh.enabled` | Enable or disable automatic refreshing of the sessions list. | `false` |
| `jules-extension.autoRefresh.interval` | The interval in seconds for auto-refreshing the session list. The minimum value is 10 seconds. | `30` |
| `jules-extension.customPrompt` | A custom prompt that is automatically prepended to every message sent to Jules. This is useful for providing persistent instructions or context (e.g., "Always reply in Japanese."). | `""` |
| `jules-extension.hideClosedPRSessions` | Automatically hide sessions associated with closed or merged pull requests from the session list to keep it tidy. | `true` |
| `jules.defaultBranch` | The default branch selection behavior when creating a Jules session.<ul><li>`current`: Use the current Git branch.</li><li>`default`: Use the repository's default branch.</li><li>`main`: Use 'main', or the repository's default if 'main' does not exist.</li></ul> | `current` |
| `jules.githubPat` | **[DEPRECATED]** Use the `Jules: Sign in to GitHub` command instead. This setting will be removed in a future version. | `""` |

## üöÄ Getting Started

1.  Press `Ctrl + Shift + P` (or `Cmd + Shift + P`) to open the Command Palette.
2.  Run **`Jules: Set Jules API Key`** and enter your API key.
3.  Click the `$(robot)` icon in the sidebar to open the Jules Sessions View.
4.  Run **`Jules: Create Jules Session`** to start your first coding session!

## Commands

This extension provides the following commands in the Command Palette (`Ctrl+Shift+P` or `Cmd+Shift+P`):

- **`Jules: Set Jules API Key`**: Configure your API credentials.
- **`Jules: Verify Jules API Key`**: Test your API connection.
- **`Jules: List Jules Sources`**: Browse available sources.
- **`Jules: Create Jules Session`**: Start a new analysis session.
- **`Jules: Refresh Jules Sessions`**: Reload the sessions list.
- **`Jules: Show Jules Activities`**: View session activities.
- **`Jules: Refresh Jules Activities`**: Update the activities view.
- **`Jules: Send Message to Jules Session`**: Post follow-up instructions to an active session.
- **`Jules: Approve Jules Plan`**: Approve a generated plan for execution.
- **`Jules: Open Jules Settings`**: Open the extension settings.
- **`Jules: Delete Session from Local Cache`**: Remove a session from the local cache.
- **`Jules: Set GitHub Token (for PR Status)`**: Securely store your GitHub token.
- **`Jules: Clear Jules Cache`**: Clear all cached data.
- **`Jules: Sign in to GitHub`**: Authenticate with GitHub using OAuth.
- **`Jules: [DEPRECATED] Set GitHub PAT`**: Deprecated method to set a GitHub Personal Access Token.
