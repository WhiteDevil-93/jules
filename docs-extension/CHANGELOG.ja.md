# 変更履歴

このファイルには、"jules-extension" 拡張機能のすべての注目すべき変更点が記録されます。

このファイルの構造については、[Keep a Changelog](http://keepachangelog.com/) の推奨事項を確認してください。

## [未リリース]

### パフォーマンス

- GitHub PRのステータスチェックを並列化し、セッションの更新パフォーマンスを向上させました。

## [1.1.2] - 2024-07-22

### 修正

- `setGitHubPat` コマンドがPATフォーマットを正しく検証していなかった問題を修正しました。

## [1.1.1] - 2024-07-21

### 変更

- GitHubトークン設定時のユーザーエクスペリエンスを向上させるため、より有益なメッセージを提供するようにしました。
- 最新の変更と改善を反映するように `README.md` を更新しました。

## [1.1.0] - 2024-07-20

### 追加

- GitHubのOAuthサインインを導入し、より安全で便利な認証方法を提供しました。
- OAuthフローを開始するための新しいコマンド `jules-extension.signInGitHub` を追加しました。
- デフォルトのブランチ選択の挙動を制御するための新しい設定 `jules.defaultBranch` を追加しました。

### 変更

- 新しいOAuthサインイン方法を優先するため、 `jules.githubPat` 設定を非推奨にしました。
- `jules-extension.setGitHubPat` コマンドが非推奨の警告を表示するようになりました。

## [1.0.9] - 2024-07-19

### 追加

- ソースとブランチのローカルキャッシュをクリアするための新しいコマンド `jules-extension.clearCache` を追加しました。
- クローズまたはマージされたプルリクエストを持つセッションを自動的に非表示にするための新しい設定 `jules-extension.hideClosedPRSessions` を追加しました。

### 変更

- ローカルキャッシュを導入することで、ソース一覧の表示パフォーマンスを向上させました。
- `listSources` コマンドがキャッシュを使用し、より高速な応答を提供するようになりました。

## [1.0.8] - 2024-07-18

### 変更

- パッケージバージョンを `1.0.8` に更新しました。
- CIとワークフロー：クロスプラットフォーム互換性とリンティングの改善。
- Geminiワークフロー：日本語出力を強制するための新しい `language` 入力を追加しました。
- 再利用性：ワークフローがGCP/Geminiの `inputs` を受け入れるようになりました。
- 出力処理：環境出力をステップおよびジョブの出力に変換するための `collect_outputs`/`collect_selected` ステップを追加しました。

### 修正

- LLMの指示の断片によって引き起こされるYAMLのパースエラーを修正しました。

### セキュリティ

- フォークコンテキストでの意図しない書き込みを避けるために、ステップを強化しました。
